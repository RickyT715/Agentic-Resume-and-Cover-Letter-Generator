<instructions>
1. **Analyze**: Review the job description to identify:
- Required technical skills and programming languages
- Preferred qualifications and experience
- Key responsibilities of the role
- Industry-specific keywords and terminology
- Desired accomplishments or impact areas
- Required years of experience (e.g., "3+ years")
2. **Select**: Then review the user's information and identify which experiences, projects, skills, and accomplishments best align with the job requirements. Choose only the experiences, projects, and skills from the user's information that are most relevant to the target role.
3. **Describe Accurately**: For each selected experience/project, describe it using ONLY facts explicitly stated in <user_information>:
 - Use the exact metrics, numbers, and results provided by the candidate
 - Describe the actual technologies and tools the candidate used
 - Reframe and reword existing accomplishments to better match JD language, but do NOT add new facts
 - Incorporate relevant technical keywords from the job description ONLY where the candidate genuinely has that skill
 - Do NOT invent metrics, percentages, scale numbers, or quantitative results not provided by the candidate
 - Do NOT add skills, technologies, or tools the candidate does not explicitly list
4. **Write**: Compose bullet points following these rules:
 - Start every bullet with a powerful action verb (Engineered, Architected, Optimized, Deployed, etc.)
 - Use at least 8 DIFFERENT action verbs across the resume (e.g., Engineered, Built, Optimized, Deployed, Implemented, Led, Reduced, Automated)
 - Use the X-Y-Z Formula: Structure every bullet point using "Accomplished [X] as measured by [Y], by doing [Z]" format:
   - X = What you accomplished
   - Y = Quantifiable metric or measurement (use ONLY real metrics from user_information)
   - Z = How you did it (methods, technologies, actions)

   Examples:
   - Instead of: "Developed a web application"
   - Write: "Built a full-stack e-commerce platform serving 10,000+ users by implementing React frontend and Node.js backend with PostgreSQL database"
 - Keep each bullet to exactly ONE line—if it spills over by 1-2 words, rewrite it shorter
 - Include specific technologies, programming languages, and tools used
5. **Format**: Output the complete resume using the exact LaTeX template provided below.
6. **Contact Info**: Use the candidate's name, email, phone, and LinkedIn **exactly as listed in <user_information>**. The LaTeX template header contains placeholders only—replace them with the real contact details from <user_information>. </instructions>




<constraints>
- **Length**: Strictly 1 page. Ruthlessly edit to fit.
- **Format**: Use ONLY the provided LaTeX template structure and commands.
- **No Fabrication**: Do NOT invent, fabricate, or estimate any information not explicitly stated in <user_information>. Do not add skills the candidate doesn't explicitly list. Do not invent metrics, percentages, or quantitative results. Do not estimate scale, user counts, or performance improvements. Every fact in the resume must be traceable to <user_information>.
- **Bullet Points**:
 - At least 3 and at most 5 bullets per experience/project
 - Each bullet must contain quantified impact when the candidate provides real metrics
 - Bullet points should be a complete 1 printed line or 2 printed line but never exceed 2 printed line
- **Skills Section**:
 - List programming languages prominently; mirror keywords from job description.
 - **CRITICAL**: Include ONLY skills/technologies from the JD that the candidate explicitly lists in their background. Do NOT add skills the candidate does not have.
 - Group skills by category (e.g., Languages, Frameworks, Tools, Cloud, Databases) and ensure JD-mentioned technologies that the candidate has appear as exact matches.
 - **Dual-form keywords**: For common abbreviations, include BOTH the abbreviation and full form on first use. Examples: "Machine Learning (ML)", "Amazon Web Services (AWS)", "Kubernetes (K8s)", "Natural Language Processing (NLP)", "Continuous Integration/Continuous Deployment (CI/CD)". This ensures matching regardless of which form the ATS looks for.
- **Summary Section**: Include a 1-2 sentence Summary section for experienced candidates (2+ years). Must contain: years of experience matching JD requirements, core technical domain, 2-3 key JD technologies, and one real achievement from the candidate's background. Omit Summary entirely for new graduates with no professional experience. If experience level instructions are provided below, follow them strictly for section ordering and Summary inclusion.
- **Keyword Distribution**: Distribute important keywords across MULTIPLE sections — not just Skills. The same key terms should appear in:
 - Summary section (keyword-dense professional overview)
 - Skills section (categorized listing)
 - Experience bullet points (demonstrating practical use)
 This cross-section reinforcement significantly improves ATS matching scores.
- **Section Headers**: Use standard, ATS-recognized section headers: "Summary", "Experience", "Education", "Skills", "Projects". Avoid creative alternatives like "What I've Done" or "My Toolbox" — ATS systems look for standard names.
- **Experience Statement**: If the JD specifies years of experience (e.g., "3+ years"), include a matching statement ONLY if the candidate actually has that experience level based on <user_information>.
- **Strict Honesty**: Do not fabricate degrees, job titles, company names, certifications, metrics, or any other information. Only use what is provided in <user_information>.
</constraints>
