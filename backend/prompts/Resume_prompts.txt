<instructions> 
1. **Analyze**: Review the job description to identify:
- Required technical skills and programming languages
- Preferred qualifications and experience
- Key responsibilities of the role
- Industry-specific keywords and terminology
- Desired accomplishments or impact areas
2. **Select**: Then review the user's information and identify which experiences, projects, skills, and accomplishments best align with the job requirements. Choose only the experiences, projects, and skills from the user's information that are most relevant to the target role. 
3. **Enhance**: For each selected experience/project, you may extend the original information by:
 - Inventing plausible quantitative results (percentages, metrics, scale) (e.g., "improved performance by 40%", "reduced latency by 200ms")
 - Elaborating on technical challenges faced and solutions implemented
 - Specifying technologies that would logically have been used even if not explicitly mentioned
 - Incorporating relevant technical keywords from the job description 
 - Adding reasonable scale metrics (dataset sizes, user counts, system throughput)
 - Keep enhancements realistic and aligned with the role level and context
4. **Write**: Compose bullet points following these rules: 
 - Start every bullet with a powerful action verb (Engineered, Architected, Optimized, Deployed, etc.) 
 - Use the X-Y-Z Formula: Structure every bullet point using "Accomplished [X] as measured by [Y], by doing [Z]" format:
   - X = What you accomplished
   - Y = Quantifiable metric or measurement
   - Z = How you did it (methods, technologies, actions)
   
   Examples:
   - Instead of: "Developed a web application"
   - Write: "Built a full-stack e-commerce platform serving 10,000+ users by implementing React frontend and Node.js backend with PostgreSQL database"
 - Keep each bullet to exactly ONE line—if it spills over by 1-2 words, rewrite it shorter
 - Include specific technologies, programming languages, and tools used 
5. **Format**: Output the complete resume using the exact LaTeX template provided below.
6. **Contact Info**: Use the candidate's name, email, phone, and LinkedIn **exactly as listed in <user_information>**. The LaTeX template header contains placeholders only—replace them with the real contact details from <user_information>. </instructions>

- **Skills Section**: List programming languages prominently; mirror keywords from job description. 
- **Honesty Boundary**: Enhancements must be plausible extrapolations—do not fabricate degrees, job titles, company names, or certifications.
